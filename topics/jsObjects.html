Javascript Objects, Inheritance, Properties, Methods, Instantiation

Objects

JavaScript is almost entirely made up of objects. Dates, math, regular expressions, arrays, and functions are always objects. Booleans, numbers and strings are “primitive” values, unless defined the the keyword “new”. A primitive value has no properties or methods and are hardcoded so they cannot be changed. Null, and undefined are “primitive” values.

Examples of primitive values:

Boolean(true);
String(“abc”);
Number(123);


Examples of objects:

new Boolean(true);
new String(“abc”);
new Number(123);


Object Creation and Properties

An object can be created with an object literal, where the property and values are defined at declaration in curly braces and separated by a comma. The properties of the javascript object can be accessed by dot notation or bracket notation

var car = {model: ‘Montero’, manufacturer: ’Mitsubishi’};
 or
var car = {
  model: ‘Montero’,
  manufacturer: ’Mitsubishi’
};
console.log(car.model); // Output Montero
console.log(car[‘manufacturer’]); //Output Mitsubishi

Additional properties can be added after creation using the dot notation

var car = {model: ‘Montero’, manufacturer: ’Mitsubishi’};
car.color = “Silver”;

Objects can also be created with a constructor

function Car(model, manufacturer) {
 this.model = model;
 this.manufacturer = manufacturer;
}
var car1 = new Car(“Montero”, “Mitsubishi”);

Object Methods

Methods can be created for an objecting using function()

car.description = function() {
 return this.manufacturer + ‘ ‘ + this.model + ‘ ‘ + this.color;
};

Object methods can be accessed with the syntax objectName.methodName();

console.log(car.description());
 // output Mitsubishi Montero Silver

Instantiation

If one wants to create more than one “instance” of an object (lets go with the above example of the car object), one should encapsulate the logic inside a function that can called whenever needed to create a new car. This is called Functional Instantiation and is basically a constructor function.

function Car (model, manufacturer, color) {
 var car = {};
 car.model = model;
 car. manufacturer = manufacturer;
 car.color = color;
 return car;
}
// Calling the function
const bigBertha = Car("Montero", "Mitsubishi", "silver")
console.log(bigBertha); //Output { model:"Montero", manufacturer:"Mitsubishi", color:"silver" }

Inheritance

In class-based object-oriented languages, Inheritance is when one object acquires all the properties and behaviors of another object. However, JavaScript is not class-based, so Javascript works on a prototype reference chain that goes in the opposite direction as traditional inheritance.

In the example below, the Animal and Rabbit classes function just fine on their own. Each class calls it's own constructor to create the object. However, since a rabbit is an animal, we might want to have access to the "run" function in Animal.

If we declare the Rabbit class with the keyword "extends" (shown below), it associates class Rabbit with class Animal. When the constructor for Rabbit is called, javaScript will take methods from the Animal class if it doesn't already exist in the Rabbit class.

class Animal {
 constructor(){...}
 run() {...}
 stop() {...}
}

class Rabbit extends Animal {
 constructor(){...}
 hide() {...}
}
